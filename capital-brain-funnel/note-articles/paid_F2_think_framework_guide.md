## THINKフレームワークとは

SCANが「情報の入力」を構造化するものなら、THINKは「判断の処理」を構造化するものだ。

どれだけ質の高い情報を集めても、判断プロセスがバイアスまみれなら正しい結論にはたどり着かない。THINKフレームワークは、AIをバイアスチェッカーとして活用し、5つのステップで投資判断を構造化する。

```
THINK フレームワーク 5ステップ

Step 1: Define  — 判断すべき問いを明確にする
Step 2: Analyze — AIで多角的に分析する
Step 3: Challenge — 反論を生成してバイアスを壊す
Step 4: Score — 判断スコアを算出する
Step 5: Decide — 感情フィルターを通して最終判断する
```

## 投資判断に潜む7大バイアス

まず、敵を知ることから始める。投資判断を歪める主要なバイアスは7つある。

### 1. 確証バイアス
自分の仮説を支持する情報ばかり集め、否定する情報を無視する。
「この銘柄は上がる」と思った瞬間から、上がる理由ばかり目に入る。

### 2. 損失回避バイアス
同額の利益と損失では、損失の方が心理的に2倍重く感じる。
含み損の銘柄を「いつか戻る」と持ち続け、損失を拡大させる。

### 3. アンカリングバイアス
最初に見た数値に判断が引きずられる。
「買値は1,000円だったから、せめて1,000円に戻るまで」と、買値に固執する。

### 4. ハーディングバイアス（群集心理）
他人の行動に追従する。
「SNSでみんなが買っている」で飛び乗り、高値掴みする。

### 5. 近時バイアス
直近の出来事を過大評価する。
先週の暴落が記憶に残り、合理的な買いのタイミングを逃す。

### 6. オーバーコンフィデンスバイアス
自分の判断力を過大評価する。
「自分は市場より賢い」と思い、集中投資やレバレッジを過度にかける。

### 7. サンクコストバイアス
すでに投じたコスト（時間・お金）に引きずられる。
「もう3ヶ月も調べたから」「ここまで下がったから」で損切りできない。

**THINKフレームワークは、この7つのバイアスを構造的に排除するために設計されている。**

## Step 1: Define — 判断すべき問いを明確にする

投資判断の第一歩は、**「何を判断するのか」を明確に言語化すること**。

多くの投資家が判断をミスる理由の一つは、そもそも「問い」が曖昧なことだ。「テスラどうかな」は問いではない。

### 判断テンプレート

```
【判断すべき問い】
[銘柄名]を、
[金額]の範囲で、
[期間]の投資期間で、
[目的: 成長/配当/分散/etc.]のために、
[買う/追加する/持ち続ける/売る] べきか？
```

例：
```
「エヌビディアを、ポートフォリオの10%（50万円）の範囲で、
 6ヶ月の投資期間で、成長狙いのために、新規購入すべきか？」
```

問いが明確になるだけで、判断の質は飛躍的に上がる。なぜなら、曖昧さがバイアスの温床になるからだ。

## Step 2: Analyze — AIで多角的に分析する

問いが定義できたら、AIを使って多角的に分析する。ここで重要なのは**一方向からだけ見ない**こと。

### プロンプト: THINK多角分析

```
以下の投資判断について、多角的に分析してください。

【判断すべき問い】
（※ Step 1で定義した問いを貼り付け）

【分析の観点】

1. ファンダメンタルズ分析
   - 財務健全性の評価
   - 成長性の評価
   - バリュエーション（割安/適正/割高）の評価

2. テクニカル分析的観点
   - 現在の株価位置（トレンドの中での位置）
   - サポート/レジスタンスの目安

3. マクロ環境との整合性
   - 現在のマクロ環境はこの投資に有利か不利か

4. ポートフォリオ観点
   - この投資を追加することのポートフォリオ全体への影響
   - 集中度リスクの評価

【各観点の評価】
観点ごとに、5段階評価 + 根拠を出力してください。

【総合判断材料サマリー】
- 投資を支持する根拠（3つ）
- 投資に反対する根拠（3つ）
```

## Step 3: Challenge — 反論を生成してバイアスを壊す

THINKフレームワークの最も重要なステップがここだ。

Step 2で分析した結果を見ると、人間は無意識に「自分が望む結論」に寄せて解釈する（確証バイアス）。これを壊すために、**AIに意図的に反論を生成させる**。

### プロンプト: デビルズ・アドボケイト（悪魔の代弁者）

```
あなたは経験豊富な反対派アナリストです。

以下の投資判断に対して、
【徹底的に反対の立場】から論じてください。

【投資判断】
（※ Step 1の問い + Step 2の分析サマリーを貼り付け）

---

以下の構造で反論してください：

【この投資が失敗するシナリオ 3つ】
各シナリオの発生確率と、発生した場合の損失見込みを含めて。

【見落としている可能性のあるリスク 3つ】
一般的な分析では見落としがちな、構造的・潜在的リスク。

【過大評価している可能性のあるポイント】
上記の分析で、楽観的すぎる部分はどこか。

【最悪のケースでの損失シミュレーション】
- 最悪ケースの具体的な数字
- その場合のポートフォリオ全体への影響

【それでもこの投資を行うなら】
- 最低限クリアすべき条件（3つ）
- 撤退ルール（この条件を満たしたら売る）
```

### なぜ反論が重要なのか

反論を読んだ上で「それでも投資する」と判断できるなら、その判断は強い。反論に対する反論が思いつかないなら、その投資は再考すべきだ。

**反論に耐えた判断だけが、実行に値する。**

## Step 4: Score — 判断スコアを算出する

感覚ではなく、定量的にスコアリングする。

### プロンプト: 投資判断スコアリング

```
以下の投資判断について、定量的なスコアリングを行ってください。

【投資判断】
（※ Step 1〜3の結果を要約して貼り付け）

---

以下の10項目について、各10点満点で採点してください。

【ファンダメンタルズ】
1. 財務健全性:    /10
2. 成長性:        /10
3. 収益性:        /10

【バリュエーション】
4. 割安度:        /10

【マクロ適合性】
5. マクロ環境の追い風:  /10

【リスク】
6. リスク管理可能性:    /10（リスクが制御可能なほど高得点）

【タイミング】
7. エントリータイミング: /10

【ポートフォリオ適合性】
8. 分散効果:            /10

【カタリスト】
9. 上方カタリストの強さ: /10

【確信度】
10. 総合確信度:          /10

---

【合計スコア】:    /100

【スコア別判断基準】
- 80〜100: 強く推奨（積極的に投資）
- 65〜79: 推奨（通常サイズで投資）
- 50〜64: 中立（小さいポジションなら可）
- 35〜49: 非推奨（見送り）
- 0〜34: 強く非推奨（投資すべきでない）

【このスコアに基づく推奨アクション】
```

### スコアの使い方

スコアは「答え」ではなく「判断材料」として使う。

- **スコア80以上**: 確信を持ってエントリー。通常〜やや大きめのポジション
- **スコア65〜79**: エントリー可。通常サイズのポジション
- **スコア50〜64**: エントリーするなら小さめ。追加調査が望ましい
- **スコア50未満**: 見送り。他の銘柄にリソースを使うべき

重要なのは、**スコアを事前に決めてから投資すること**。スコアが低いのに「でもなんとなく良さそう」で投資するなら、スコアリングの意味がない。

## Step 5: Decide — 感情フィルターを通して最終判断する

最後のステップは、**自分の感情状態をチェックする**こと。

どれだけ優れた分析をしても、判断の瞬間に感情が入れば台無しになる。

### 感情フィルター チェックリスト

以下の質問に正直に答える（自問自答 or AIに整理を手伝ってもらう）。

```
【感情フィルター チェックリスト】

□ 1. この判断を急いでいないか？
     （FOMOや焦りで動いていないか）

□ 2. 最近の損失/利益に引きずられていないか？
     （前の取引の結果が今回の判断に影響していないか）

□ 3. 他人の意見に影響されていないか？
     （SNS、YouTube、知人の助言で判断が揺らいでいないか）

□ 4. 「取り返したい」という気持ちがないか？
     （リベンジトレードの兆候はないか）

□ 5. この投資を1年後の自分が見ても「合理的だ」と思えるか？
     （短期的な感情ではなく、長期的な視点で納得できるか）
```

### 最終判断のルール

```
判断基準:
  スコア65以上 ＋ 感情フィルター全項目クリア → 実行
  スコア65以上 ＋ 感情フィルター1つ以上NG → 1日待って再評価
  スコア65未満 → 見送り（理由を記録して次へ）
```

**「1日待つ」ルールが最も強力なバイアス対策だ。** 感情は24時間で大幅に薄れる。翌日に同じ判断ができるなら、それは感情ではなく論理に基づいている。
